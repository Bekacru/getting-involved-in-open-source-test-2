Git

Git is a version control system that allows us to track changes, collaborate with other, undo mistakes and manage code history.

Common Git Operations

Initializing

Initialize a new git repository. It'll create a hidden .git folder that'll hold our code history.

git init

Staging

Prepares our changes to be committed. We use "." to specify to stage the whole directory.

git add .

Commit

Commits the changes we made to the git history. And we use `-m [message]` to specify a commit message.

git commit -m [commit message]

Add Remote

Let's add a remote repository to our local git project.

git remote add [remote name] [remote address]

Branches

A separate lines of development in one git repository.

git branch [branch name] # creates a new branch git checkout [branch name] # switch to a branch

Push

push changes to a remote repository.

git push

Pull

pulls changes from a remote repository to our local repo.

git pull

Merge

Combines changes from one branch into another by creating a new merge commit.

git merge

Rebase

It rearranges the commit history of the branch you're currently on to make it look like it was started from the most recent commit of another branch.

git rebase
